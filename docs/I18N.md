# SFTP+ 国际化指南

> 创建人：DD1024z + Deepseek-V4-Flash
> 创建时间：2026-06-25

## 概述

SFTP+ 使用自建的轻量级国际化系统（不依赖 Angular i18n 或 ngx-translate），由 `SftpI18nService` 统一管理。全部翻译定义在 `src/sftp-i18n.service.ts` 的 `TRANSLATIONS` 常量中。

## 支持的语言

| 语言代码 | 语言 | 状态 |
|---------|------|------|
| `zh-CN` | 简体中文 | 完整支持 |
| `en-US` | English | 完整支持 |

## 语言检测策略（五级回退）

```
优先级 1 (最高)  →  用户手动选择（localStorage: 'sftp-plus-locale'）
优先级 2        →  设置页保存的语言（localStorage: 'sftp-plus-settings.lang'）
优先级 3        →  Tabby ConfigService（Angular DI 注入）
优先级 4        →  navigator.languages（浏览器首选语言列表）
优先级 5        →  navigator.language（浏览器单一语言设置）
优先级 6 (最低)  →  默认值 'zh-CN'
```

## 使用方法

### 在 TypeScript 代码中

```typescript
// 简单文本
this.i18n.t('pane.local');       // → "本地" (zh-CN) / "Local" (en-US)

// 带参数替换
this.i18n.t('pane.items', { count: 42 });
// → "42 项" (zh-CN) / "42 items" (en-US)

// 在模板中
// <span>{{ i18n.t('pane.local') }}</span>
// <span>{{ i18n.t('pane.nItems', {count: items.length}) }}</span>

// 动态拼接
this.i18n.t('file.kb', { size: '128' });
// → "128 KB" (zh-CN) / "128 KB" (en-US)
```

### 参数替换语法

在翻译字符串中使用 `{key}` 占位符：

```json
{
  "zh-CN": {
    "bookmark.deleteConfirm": "确定要删除这 {count} 个项目吗？"
  },
  "en-US": {
    "bookmark.deleteConfirm": "Delete {count} item(s)?"
  }
}
```

调用时传入参数对象：`i18n.t('bookmark.deleteConfirm', { count: 3 })`

## 新增翻译

### 添加新键

1. 在 `src/sftp-i18n.service.ts` 的 `TRANSLATIONS` 中找到对应的语言区块
2. 在所有支持的语言中添加相同的键名
3. 注意保持键名在两种语言中一致

示例：

```typescript
private readonly TRANSLATIONS: Record<Locale, Record<string, string>> = {
  'zh-CN': {
    // ... 已有翻译
    'app.undo': '撤销',
  },
  'en-US': {
    // ... 已有翻译
    'app.undo': 'Undo',
  },
};
```

### 添加新语言

1. 在 `Locale` 类型中添加新语言代码
2. 在 `TRANSLATIONS` 中添加新语言区块
3. 在语言检测策略中增加对该语言的识别
4. 在设置页的语言选择下拉中添加新选项

```typescript
type Locale = 'zh-CN' | 'en-US' | 'ja-JP';  // 新增日语

private readonly TRANSLATIONS: Record<Locale, Record<string, string>> = {
  'zh-CN': { /* ... */ },
  'en-US': { /* ... */ },
  'ja-JP': { /* ... */ },  // 新增
};
```

**重要：** 新语言必须包含**所有已有翻译键**的对应翻译，不可遗漏。如果某个键暂时没有翻译，系统会自动回退到 `en-US`。

## 回退机制

当翻译键缺失时，系统按以下顺序回退：

```
目标语言 → en-US → 返回键名本身
```

例如，`ja-JP` 中缺少 `app.undo`：
```
ja-JP: app.undo → 不存在
en-US: "Undo"  → 存在，使用
```

如果 `en-US` 也缺失，则直接返回 `'app.undo'`（键名本身）。

## 最佳实践

1. **全部文本国际化**：所有用户可见的字符串都应使用 `i18n.t()`，包括错误消息、提示文本、按钮标签
2. **避免字符串拼接**：使用参数模板 `{key}`，而不是在代码中拼接
3. **保持键名一致**：同一含义的文本在不同地方应使用相同的键
4. **语义化键名**：使用 `notify.transferComplete` 而非 `notify.msg1`
5. **考虑文本长度**：某些语言（如中文）比英文更短，UI 布局应能适应不同长度的文本
6. **添加注释**：如果翻译上下文不明显，在键名旁边添加注释说明使用场景
